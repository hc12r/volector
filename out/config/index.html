Configuration Guide

Configure Volector sources via the YAML catalog

Overview
The CLI command run selects a source by name and country and crawls its base_urls

Default catalog path: crawler/config/sources.yaml

Example Configuration
yaml
1sources:
2  - name: example
3    base_urls: ["https
4    country: MZ
5    language: pt
6    render: false
7    allow: ["/news/"]
8    deny: ["/login", "/privacy"]
9    schedule: "0 * * * *"   # hourly
10    max_pages: 200
Configuration Fields
Requiredname

Unique source identifier (string)

Requiredbase_urls

One or more entry URLs to fetch (list of strings)

Requiredcountry

Country code (e.g., MZ, US)

Optionallanguage

Language code (e.g., pt, en)

Optionalrender

If true, indicates preference for JS rendering (boolean)

Optionalallow

Path prefixes to include (list of strings)

Optionaldeny

Path prefixes to exclude (list of strings)

Optionalschedule

5-field cron expression (minute hour day month day_of_week)

Optionalmax_pages

Upper bound of base_urls to process in a single run (integer)

Cron Schedule Format
"0 * * * *"→ run at minute 0 of every hour
"30 2 * * *"→ run daily at 02:30
"*/15 * * * *"→ run every 15 minutes
Environment Variables
bash
1# MinIO Configuration
2export MINIO_ENDPOINT=localhost:9000
3export MINIO_ACCESS_KEY=your-access-key
4export MINIO_SECRET_KEY=your-secret-key
5export MINIO_BUCKET=volector-data
6
7# Storage paths
8# Curated Parquet files stored under:
9# curated/{entity}/{country}/dt=YYYY-MM-DD/data.parquet
Best Practices
  • Start small: a few base URLs and a low max_pages to validate the pipeline
  • Respect robots.txt - Volector checks robots for each URL before fetching
  • Prefer static fetching; only enable JS rendering for pages that need it
  • Use meaningful source names for easier management
