Contributing Guide

Help make Volector better - contribute code, documentation, or feedback

Getting Started
Set up your development environment

1. Fork and Clone

bash
1# Fork the repository on GitHub, then clone your fork
2git clone https://github.com/YOUR_USERNAME/volector.git
3cd volector
4
5# Add upstream remote
6git remote add upstream https://github.com/hc12r/volector.git

2. Development Setup

bash
1# Create virtual environment
2python -m venv venv
3source venv/bin/activate  # On Windows: venv\Scripts\activate
4
5# Install in development mode
6pip install -e ".[dev,test,all]"
7
8# Install pre-commit hooks
9pre-commit install

3. Run Tests

bash
1# Run the full test suite
2pytest
3
4# Run with coverage
5pytest --cov=crawler --cov-report=html
6
7# Run specific test categories
8pytest -m "not integration"  # Skip integration tests
9pytest tests/unit/           # Unit tests only
Development Workflow

1. Create Feature Branch

bash
1git checkout -b feature/your-feature-name
2# or
3git checkout -b fix/issue-description

2. Make Changes

Follow the coding standards and add tests for new functionality.

bash
1# Run linting and formatting
2black crawler/ tests/
3isort crawler/ tests/
4flake8 crawler/ tests/
5
6# Run tests frequently
7pytest tests/unit/test_your_module.py

3. Commit Changes

bash
1# Stage your changes
2git add .
3
4# Commit with descriptive message
5git commit -m "feat: add custom pipeline support
6
7- Add BasePipeline abstract class
8- Implement pipeline registration system
9- Add tests for pipeline functionality
10- Update documentation"

4. Submit Pull Request

bash
1# Push to your fork
2git push origin feature/your-feature-name
3
4# Create pull request on GitHub
5# Include description of changes and link to issues
Coding Standards
StyleCode Formatting
  • • Use Black for code formatting (line length: 88)
  • • Use isort for import sorting
  • • Follow PEP 8 style guidelines
  • • Use type hints for all public functions
TestingTest Requirements
  • • Write tests for all new functionality
  • • Maintain >90% test coverage
  • • Use pytest fixtures for common setup
  • • Mock external dependencies in unit tests
DocumentationDocumentation Standards
  • • Use Google-style docstrings
  • • Update README.md for new features
  • • Add examples for public APIs
  • • Update changelog for breaking changes
Types of Contributions

🐛 Bug Fixes

Fix issues, improve error handling, or resolve edge cases. Include reproduction steps and tests.

✨ New Features

Add new crawling capabilities, storage backends, or pipeline stages. Discuss large features first.

📚 Documentation

Improve guides, add examples, fix typos, or create tutorials for common use cases.

🧪 Testing

Add test cases, improve test coverage, or create integration tests for real-world scenarios.

⚡ Performance

Optimize crawling speed, reduce memory usage, or improve storage efficiency.
